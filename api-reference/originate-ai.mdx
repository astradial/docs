## What is originate to AI?

This endpoint calls someone (an extension or external phone number) and when they answer, connects them directly to an AI agent running on Asterisk Stasis.

Think of it like this: your AI bot picks up the phone, dials a number, and starts talking when the person answers.

<Info>
The call enters a Stasis application where the AI agent controls the conversation via WebSocket. No dialplan routing is involved.
</Info>

## How it works

<Steps>
  <Step title="Your system sends the API request">
    You call the originate-to-ai endpoint with the destination number and AI agent details.
  </Step>
  <Step title="Asterisk dials the number">
    The phone system calls the person. They see your caller ID on their phone.
  </Step>
  <Step title="Person answers">
    When they pick up, the call is immediately connected to your AI agent via WebSocket.
  </Step>
  <Step title="AI agent takes over">
    Your AI agent (OpenAI, Deepgram, custom, etc.) handles the conversation in real-time.
  </Step>
</Steps>

## Key features

- Direct call origination to Stasis app (no dialplan routing)
- WebSocket URL support for external AI services
- Supports both internal extensions and external numbers
- Channel variables passed to AI agent handler
- Async call origination (non-blocking)

## Use cases

- AI-powered outbound calling campaigns
- Automated appointment reminders with conversational AI
- Customer surveys with natural language processing
- Voice assistant integrations (OpenAI Realtime API, Deepgram, etc.)
- Real-time speech-to-speech translation services

## Endpoint

```
POST /api/v1/calls/originate-to-ai
```

### Authentication

Include your API key or token in the header:

```
Authorization: Bearer YOUR_ORG_TOKEN
```

or

```
X-API-Key: ak_your_api_key
```

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `to` | string | Yes | Extension number (e.g. `2001`) or external phone number (e.g. `+919944421125`) |
| `caller_id` | string | No | Name or number shown on the recipient's phone |
| `wss_url` | string | No | WebSocket URL for external AI service |
| `ai_agent_app` | string | No | Stasis application name (default: `ai_agent`) |
| `timeout` | number | No | Ring timeout in seconds (default: 30) |
| `variables` | object | No | Custom channel variables passed to the AI agent |

## Examples

### Call an extension with an AI agent

<CodeGroup>
```bash curl
curl -X POST https://devpbx.astradial.com/api/v1/calls/originate-to-ai \
  -H "Authorization: Bearer YOUR_ORG_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "2001",
    "caller_id": "AI Assistant",
    "wss_url": "wss://ai.example.com/voice/stream"
  }'
```

```javascript node.js
const response = await fetch("https://devpbx.astradial.com/api/v1/calls/originate-to-ai", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_ORG_TOKEN",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    to: "2001",
    caller_id: "AI Assistant",
    wss_url: "wss://ai.example.com/voice/stream"
  })
});
```

```python python
import requests

response = requests.post(
    "https://devpbx.astradial.com/api/v1/calls/originate-to-ai",
    headers={
        "Authorization": "Bearer YOUR_ORG_TOKEN",
        "Content-Type": "application/json"
    },
    json={
        "to": "2001",
        "caller_id": "AI Assistant",
        "wss_url": "wss://ai.example.com/voice/stream"
    }
)
```
</CodeGroup>

```json
  "channel_id": "1713045600.123"
}
```

## Common errors

| Status | Error | Cause |
|--------|-------|-------|
| 400 | `to is required` | Missing the `to` field |
| 401 | `Authentication required` | Missing or invalid API key |
| 500 | `Origination failed` | Asterisk could not place the call |

<Warning>
The `to` field must be either an internal extension number or a full phone number with country code (e.g. `+919944421125`). The caller must have permission to call external numbers if dialing out.
</Warning>

<Tip>
Use the `variables` field to pass context to your AI agent. For example, pass the customer's name, booking ID, or language preference so the bot can personalize the conversation.
</Tip>
