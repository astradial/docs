## What are API keys?

An API key is like a password that lets other software talk to your Astradial account. When your CRM, website, or any other tool needs to make calls, pull call logs, or trigger workflows through Astradial, it uses an API key to prove it has permission.

Think of it like a key card for a hotel room. The hotel (Astradial) gives you a key card (API key), and every time you want to get into your room (use the API), you swipe the card to prove you are allowed in.

<Info>
  You only need API keys if you are building integrations or using the Astradial API from your own software. If you only use the dashboard at [editor.astradial.com](https://editor.astradial.com), you do not need this.
</Info>

## How to create an API key

<Steps>
  <Step title="Open the API page">
    In the left sidebar, click **Automate**, then click **API & Webhooks**.
  </Step>

  <Step title="Go to the API Keys tab">
    Click the **API Keys** tab at the top of the page.

    You will see a list of all API keys for your organisation. If you have not created any yet, the list will be empty.
  </Step>

  <Step title="Click Create API Key">
    Click the **+ Create API Key** button. A form will open.
  </Step>

  <Step title="Name your key and set permissions">
    Fill in:

    - **Name** -- a label so you remember what this key is for (e.g. "CRM Integration", "Website Click-to-Call", "Reporting Dashboard")
    - **Permissions** -- select what this key is allowed to do

    <Tip>
      Give each integration its own API key with only the permissions it needs. This way, if a key is ever compromised, the damage is limited.
    </Tip>
  </Step>

  <Step title="Save and copy your credentials">
    Click **Create**. You will now see two values:

    - **API Key**: `ak_xxxxxxxxxxxx`
    - **API Secret**: `as_xxxxxxxxxxxx`

    <Warning>
      **Copy the API Secret right now.** It is only shown once. After you close this screen, you can never see the secret again. If you lose it, you will have to create a new key.
    </Warning>
  </Step>
</Steps>

## Using your API key

Every request you make to the Astradial API must include your API key in the headers. Here is how.

### Add the header

Include the `X-API-Key` header in every request:

```
X-API-Key: ak_your_key_here
```

### Example request

<CodeGroup>

```bash cURL
curl -X GET https://devpbx.astradial.com/api/v1/calls \
  -H "X-API-Key: ak_your_key_here"
```

```javascript JavaScript
const response = await fetch('https://devpbx.astradial.com/api/v1/calls', {
  headers: {
    'X-API-Key': 'ak_your_key_here'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://devpbx.astradial.com/api/v1/calls',
    headers={'X-API-Key': 'ak_your_key_here'}
)

data = response.json()
```

</CodeGroup>

## Types of API keys

Astradial has two types of API keys, each designed for different use cases:

| Type | What it is for | Example use |
| --- | --- | --- |
| **API Key** | Managing calls -- making calls, listing call logs, controlling live calls | CRM integration that shows call history |
| **Workflow Key** | Triggering workflows from external systems | Website form that triggers a callback workflow |

<Note>
  When you create a key, you choose which type it is. A single key cannot be both types. If you need both call management and workflow triggering, create two separate keys.
</Note>

## Best practices

<CardGroup cols={2}>
  <Card title="One key per integration" icon="layer-group">
    Do not share one key across multiple tools. If you have a CRM integration and a reporting tool, create separate keys for each.
  </Card>
  <Card title="Never expose keys in frontend code" icon="eye-slash">
    API keys should only be used in server-side code. Never put them in JavaScript that runs in the browser -- anyone could see them.
  </Card>
  <Card title="Rotate keys regularly" icon="arrows-rotate">
    Delete old keys and create new ones periodically. If a key might have been leaked, delete it immediately and create a replacement.
  </Card>
  <Card title="Use minimum permissions" icon="shield-halved">
    Only grant the permissions each key actually needs. A reporting tool does not need permission to make calls.
  </Card>
</CardGroup>
